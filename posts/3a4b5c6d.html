基于 VS Code + WSL + Conda 的可移植 Python 开发全流程：以溶菌酶数据集分析为例

发表于2025-07-01|更新于2025-07-01|开发工具Python
|浏览量:|评论数:

在 Python 开发中，跨平台环境一致性和项目迁移是常见需求。本教程以一个实际项目—— 溶菌酶数据集分析（lysozyme_dataset） 为例，带你从零开始，使用 VS Code、WSL 和 Conda 构建可移植的 Python 开发环境。你将在 Windows 上开发用于分析溶菌酶数据的 Python 脚本，并将项目无缝迁移到远程 Linux 服务器运行。每个步骤都围绕该项目展开，命令附有详细解释，适合新手和有经验的开发者！

项目背景与教程目标

项目背景：
lysozyme_dataset 是一个数据分析项目，旨在使用 Python 分析溶菌酶（lysozyme）相关的实验数据集。项目需要依赖 pandas 进行数据处理，未来可能扩展到机器学习（如 scikit-learn）。开发在 Windows 上基于WSL进行，但需要在高性能 Linux 服务器上运行批处理任务或进行后续的深度学习训练。

教程目标：

  • 在 Windows 的 WSL 环境中使用 VS Code 开发 lysozyme_dataset 项目。
  • 创建可移植的 Conda 环境，确保本地和服务器环境一致。
  • 将项目（代码 + 环境）迁移到远程 Ubuntu 服务器运行分析脚本。

适用场景：

  • 数据科学项目需要在本地开发、远程运行。
  • 跨平台开发（Windows/Linux）要求环境一致。
  • 项目需隔离环境并便于迁移或团队共享。

准备工作：

  • 本地环境：
    • 操作系统： Windows 10/11，已启用 WSL 功能。
    • WSL 发行版： 通过 Microsoft Store 安装 Ubuntu（推荐 Ubuntu 20.04 或更高）。
    • 开发工具： 已安装 Visual Studio Code。
    • VS Code 扩展：
      • Remote Development（Microsoft）：包含 Remote - WSL 和 Remote - SSH，用于远程开发。
      • Python（Microsoft）：提供 Python 代码补全、调试等功能。
    • Conda： 在 WSL 的 Ubuntu 中安装 Miniforge3（推荐，占用空间小）。运行 conda --version 检查安装。
  • 项目位置： Windows D 盘的空文件夹 D:\Research\phalp\lysozyme_dataset，用于存储项目代码和环境。
  • 远程环境： 一台可通过 SSH 和 XFTP 访问的 Ubuntu 服务器，IP 地址已知，无需预装 Python 或 Conda。

注意事项：

  • 运行 wsl --list 检查 Ubuntu 是否为默认发行版，若不是，运行 wsl --set-default Ubuntu 设置。

  • Miniforge3 安装命令：

  • 确保服务器网络稳定，SSH 和 XFTP 端口开放。

教程结构

  1. 阶段一：本地初始化与开发
    • 配置 VS Code 和 WSL，初始化 lysozyme_dataset 项目。
    • 创建或迁移 Conda 环境。
    • 配置 VS Code 自动化。
    • 开发溶菌酶数据分析脚本。
  2. 阶段二：打包与迁移
    • 打包项目环境。
    • 传输到远程服务器。
  3. 阶段三：远程服务器部署
    • 解压和修复环境。
    • 运行分析脚本。
  4. 阶段四：总结与进阶
    • 流程回顾。
    • 进阶技巧（Git、Remote SSH）。

阶段一：本地初始化与开发

第 1 步：通过 VS Code 连接 WSL 并打开 lysozyme_dataset

  1. 打开 VS Code，点击左下角的蓝色“**><**”图标（远程连接）。
  2. 选择 “Connect to WSL”，VS Code 切换到 WSL 的 Ubuntu 环境，左下角显示 WSL: Ubuntu。
  3. 点击 “File > Open Folder”，导航到 /mnt/d/Research/phalp/lysozyme_dataset（WSL 挂载的 D 盘路径），确认打开。
  4. 验证标题栏显示 [WSL: Ubuntu] lysozyme_dataset，表示项目已加载。

命令解释：

  • /mnt/d/：WSL 将 Windows D 盘挂载到 Linux 的 /mnt/d/，允许访问 Windows 文件。
  • Connect to WSL：通过 Remote - WSL 扩展，VS Code 在 Ubuntu 环境中运行，适合 Linux 兼容的开发。

项目关联：

  • lysozyme_dataset 文件夹是项目的根目录，存放代码（如 main.py）、环境（env/）和配置（.vscode/）。
  • WSL 确保本地开发与服务器环境一致，适合后续迁移。

注意事项：

  • 若连接失败，运行 wsl --status 检查 WSL 服务，或重启：wsl --shutdown。
  • 确认文件夹权限：ls -l /mnt/d/Research/phalp/。

第 2 步：为 lysozyme_dataset 创建或迁移 Conda 环境

为何将环境放在项目内？

  • 便携性： 环境随 lysozyme_dataset 文件夹移动，复制即可迁移。
  • 隔离性： 确保溶菌酶项目依赖（如 pandas）不与其他项目冲突。

选择以下方案：

方案 A：为新项目创建环境（推荐）

  1. 按 `Ctrl + `` 打开 VS Code 集成终端（WSL 的 Bash）。

  2. 确认路径：

    命令解释：

    • pwd：打印当前工作目录，应为 /mnt/d/Research/phalp/lysozyme_dataset。

  3. 创建 Python 3.10 环境：

    命令解释：

    • conda create：创建新的 Conda 虚拟环境。
    • --prefix ./env：将环境存储在 ./env（项目目录下），非全局路径。
    • python=3.10：安装 Python 3.10，适合数据分析。
    • -y：自动确认提示。

  4. 验证环境：ls，确认出现 env/ 目录。

项目关联：

  • env/ 是 lysozyme_dataset 的专用环境，包含 Python 3.10，后续安装 pandas 等依赖。
  • 项目内环境确保迁移时所有依赖随行。

注意事项：

  • 可选 Python 版本（如 python=3.9），但需与 pandas 兼容。

  • 加速下载：

  • 检查环境：conda info --envs，应列出 ./env。

方案 B：迁移现有全局环境

若已有全局环境（如 phalp），迁移到 lysozyme_dataset：

  1. 打包环境：

    命令解释：

    • conda pack：打包 Conda 环境为 tar.gz 文件。
    • -n phalp：指定全局环境名称。
    • -o phalp-env.tar.gz：输出压缩包。

  2. 移动并解压：

    命令解释：

    • mv：移动压缩包。
    • mkdir -p env：创建 env 目录。
    • tar -xzf：解压到 env，-C 指定目标目录。

  3. 修复路径：

    命令解释：

    • conda-unpack：更新环境路径，适配项目目录。

  4. （可选）删除旧环境：

    命令解释：

    • conda remove --all：删除全局环境，释放空间。

项目关联：

  • 迁移后的 env/ 成为 lysozyme_dataset 的本地环境，保持与原 phalp 相同的依赖。

注意事项：

  • 安装 conda-pack：conda install -c conda-forge conda-pack.
  • 确保 env 为空：rm -rf env/*。
  • 验证：./env/bin/python --version。

第 3 步：配置 VS Code 自动化

  1. 创建配置文件夹：

    命令解释：

    • mkdir：创建 .vscode 目录，存储 VS Code 配置。

  2. 创建 settings.json：

    命令解释：

    • touch：创建空文件。

  3. 编辑 settings.json：

    配置解释：

    • python.defaultInterpreterPath：指定 lysozyme_dataset 的 Python 解释器。
    • python.terminal.activateEnvironment：自动激活 env 环境。
    • ${workspaceFolder}：动态指向项目根目录。

  4. 重载 VS Code：code . 或点击“Reload Window”。

  5. 验证右下角显示 ('env': conda)，终端提示 (env)。

项目关联：

  • 配置确保 VS Code 识别 lysozyme_dataset 的 env，为溶菌酶数据分析提供代码补全和调试支持。

注意事项及可选设置：

  • 添加格式化配置：

  • 手动激活测试：./env/bin/conda activate。

  1. 创建 main.py：

    命令解释：

    • touch：创建 Python 脚本文件。

  2. 编写代码：

    代码解释：

    • 模拟溶菌酶数据集，包含样本 ID、浓度和活性。
    • 使用 pandas 创建 DataFrame，打印预览。

  3. 安装 pandas：

    命令解释：

    • pip install：安装 pandas 到 env 环境。

  4. 运行脚本：

    命令解释：

    • python main.py：执行脚本，验证环境和数据处理功能。

项目关联：

  • main.py 是 lysozyme_dataset 的核心脚本，展示如何处理溶菌酶数据。
  • 输出验证环境正确，未来可扩展为复杂分析（如统计或建模）。

注意事项：

  • 确认 pip 路径：which pip 应为 ./env/bin/pip。

  • 记录依赖：

  • 调试：使用 VS Code 的“Run > Start Debugging”。

阶段二：打包与迁移

第 5 步：打包 lysozyme_dataset 环境

  1. 检查是否已安装 conda-pack：

    如果未安装，则执行：

    命令解释：

    • conda list：列出已安装的包。
    • grep conda-pack：过滤查找 conda-pack。
    • conda install：安装打包工具。
    • -c conda-forge：使用 conda-forge 频道。

  2. 打包环境：

    命令解释：

    • conda pack --prefix ./env：打包项目内的 env。
    • --output env.tar.gz：生成压缩包。

  3. 验证：

    命令解释：

    • ls -lh：检查压缩包大小和创建时间。

项目关联：

  • env.tar.gz 包含 pandas 等依赖，是 lysozyme_dataset 的环境快照，准备迁移到服务器。

注意事项：

  • 检查环境大小：du -sh env。
  • 验证压缩包：tar -tvf env.tar.gz | head。
  • 确保依赖完整，如 pandas。

第 6 步：迁移到远程服务器

  1. 打开 XFTP，连接服务器（输入 IP、用户名、密码）。
  2. 左侧导航到 D:\Research\phalp\lysozyme_dataset，右侧到 /home/your_user/deploy/。
  3. 拖拽 lysozyme_dataset 文件夹到服务器。

传输内容：

  • 必须： main.py, .vscode/, env.tar.gz。
  • 可选： env/（可省略，服务器解压生成）。

项目关联：

  • 传输确保 lysozyme_dataset 的代码和环境到达服务器，用于远程数据分析。

注意事项：

  • 检查服务器空间：df -h。
  • 断点续传：XFTP 支持重传。
  • 记录路径：/home/your_user/deploy/lysozyme_dataset。

阶段三：远程服务器部署

第 7 步：解压并修复环境

  1. SSH 登录：

    命令解释：

    • ssh：远程登录服务器。

  2. 进入项目目录：

    命令解释：

    • cd：切换到项目目录。

  3. 解压环境：

    命令解释：

    • mkdir -p env：创建空 env 目录。
    • tar -xzf：解压到 env。

  4. 修复路径：

    命令解释：

    • conda-unpack：适配服务器路径。

项目关联：

  • 解压后的 env 恢复 lysozyme_dataset 的环境，支持 main.py 运行。

注意事项：

  • 清空 env：rm -rf env/*。
  • 验证解压：ls env/bin。
  • 检查压缩包：gzip -t env.tar.gz。

第 8 步：运行溶菌酶分析脚本

  1. 激活环境：

    命令解释：

    • source：加载环境变量，激活 env。

  2. 运行脚本：

    命令解释：

    • python main.py：执行分析脚本。

  3. 验证输出，确认 sys.executable 指向服务器路径（如 /home/your_user/deploy/lysozyme_dataset/env/bin/python）。

项目关联：

  • 脚本输出溶菌酶数据集预览，证明迁移成功，服务器可运行分析任务。

注意事项：

  • 确认 Python 路径：which python。
  • 安装缺失依赖：pip install <package>。
  • 检查资源：top 或 htop。
