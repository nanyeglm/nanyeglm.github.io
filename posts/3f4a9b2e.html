配置FRP实现ubuntu远程SSH访问

发表于2024-12-28|更新于2024-12-28|Linux网络工具
|浏览量:|评论数:

当我们需要远程访问位于内网或防火墙后的 Ubuntu 服务器时，往往会遇到无法直接连接的问题。FRP（Fast Reverse Proxy）提供了一个优雅的解决方案，通过反向代理的方式，安全地将内网服务暴露到公网。本文将详细介绍如何配置 FRP 来实现 Ubuntu 服务器的远程 SSH 访问。

1. FRP 基础介绍

1.1 什么是 FRP？

FRP（Fast Reverse Proxy）是一款开源的反向代理应用，旨在帮助用户将位于内网或防火墙后的服务（如 Web 服务、SSH 等）暴露到公网。它支持多种协议，并提供高效的性能和丰富的功能，适用于个人用户和企业级应用。

1.2 FRP 的工作原理

FRP 采用客户端-服务端架构：

  • 服务端（frps）：部署在拥有公网 IP 的服务器上，负责接收来自客户端的连接请求。
  • 客户端（frpc）：部署在内网环境中，连接到服务端，并将内网服务（如 SSH）通过服务端进行转发。

这种方式避免了内网服务器直接暴露在公网，提高了安全性，同时简化了内网穿透的过程。

1.3 FRP 的优势

  • 高性能：采用高效的网络传输协议，支持多种代理类型。
  • 易于配置：配置文件简洁明了，适合新手快速上手。
  • 灵活性：支持多种场景，如 HTTP/HTTPS、TCP、UDP 等。
  • 安全性：支持多种认证方式，确保连接的安全。

2. 演示环境

在开始配置之前，我们需要了解当前的环境设置：

  • 操作系统：Ubuntu（适用于其他 Linux 发行版，步骤类似）
  • 账户信息：普通用户（例如 user），无需 root 权限
  • 网络信息：
    • 内网 IP：192.168.1.10（部署 frpc 的内网机器）
    • FRP 服务器地址：example.com（拥有公网 IP 的服务器，用于部署 frps）
  • 工具：wget, tar, systemd（Ubuntu 默认已安装）

3. 配置 FRP 服务端（frps）

3.1 下载并解压 FRP 服务端

首先，在拥有公网 IP 的服务器（example.com）上下载并解压 FRP 服务端程序。假设你以普通用户身份登录，无需使用 sudo。

详细说明：

  • cd ~：切换到当前用户的主目录，确保下载文件存放在用户目录下，避免权限问题。
  • wget [URL]：从指定的 URL 下载 FRP 压缩包。可以在 FRP Releases 页面找到最新版本。
  • tar -xzvf [文件名]：解压 .tar.gz 格式的压缩包。参数说明：
    • -x：解压
    • -z：通过 gzip 进行解压
    • -v：显示详细的解压过程
    • -f：指定要解压的文件
  • cd [目录]：进入解压后的 FRP 目录，后续配置和运行需要在此目录下进行。

    • 可选配置，用于支持 HTTP 类型的代理。

  • 端口选择：bindPort 和 webPort 需要选择未被占用的端口，避免端口冲突。
  • 安全性：使用复杂的 auth.token 以防止未经授权的访问。
  • 防火墙设置：确保选择的端口在服务器的防火墙（如 ufw 或 iptables）中已开放。

为了让 FRP 服务端在服务器重启后自动启动，我们可以使用 systemd 来创建一个服务单元。

在编辑器中粘贴以下内容：

配置说明：

  • [Unit] 区块：
    • Description：服务描述。
    • After=network.target：确保网络服务启动后再启动 FRP 服务端。
  • [Service] 区块：
    • Type=simple：指定服务类型为简单类型。
    • ExecStart：指定启动服务的命令及参数。确保路径与实际文件位置一致。
    • Restart=always：当服务异常退出时，总是自动重启服务。
    • RestartSec=5：服务重启前的等待时间（秒）。
  • [Install] 区块：
    • WantedBy=multi-user.target：将服务挂载到多用户运行级别，确保在系统启动时自动启动。

注意事项：

  • 路径配置：确保 ExecStart 中的路径正确。如果 FRP 解压在其他位置，请相应调整。
  • 权限问题：如果 FRP 需要特定的权限，请确保 systemd 服务文件中的用户和权限配置正确。

4. 配置 FRP 客户端（frpc）

接下来，在内网机器（IP：192.168.1.10）上配置 FRP 客户端，以将本地的 SSH 服务通过 FRP 暴露到公网。

4.1 配置 frpc.toml 文件

首先，在内网机器上创建并编辑 FRP 客户端的配置文件 frpc.toml。

在编辑器中输入以下内容：

配置说明：

  • serverAddr = "example.com"：
    • 指定 FRP 服务端的地址，即你配置的 example.com 服务器。
  • serverPort = 7000：
    • 指定 FRP 服务端的监听端口，需与服务端 frps.toml 中的 bindPort 保持一致。
  • auth.method = "token" 和 auth.token = "your-token"：
    • 认证配置，需与服务端保持一致，确保客户端能够成功连接到服务端。
  • [[proxies]] 区块：
    • 每个 [[proxies]] 表示一个代理配置。
    • name = "ssh"：为该代理指定一个名称，便于识别和管理。
    • type = "tcp"：指定代理类型为 TCP，适用于 SSH 等基于 TCP 的服务。
    • localIP = "127.0.0.1"：指定本地服务的 IP 地址，通常为 127.0.0.1 表示本机。
    • localPort = 22：指定本地服务的端口，SSH 默认使用 22 端口。
    • remotePort = 6000：指定服务端对外暴露的端口，通过此端口可访问内网的 SSH 服务。

注意事项：

  • 端口选择：remotePort 需选择服务端未被占用的端口，确保不会与其他代理冲突。
  • 安全性：确保 auth.token 保持秘密，避免被他人获取后滥用。
